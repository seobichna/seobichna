Information Architecture

User Needs

01. 다양한 기획 과정과 결과물을 볼 수 있는가?

포토폴리오 프로세스 과정을 별도 페이지로 구분하여 기획 능력을 어필하고
워크페이지 내에 기획 및 코딩 작업물들을 대분류로 나누어 다양하게 전시하였습니다.

  • HEADER포트폴리오 버튼 별도 노출하여 바로 이동

  • INDEX SECTION - WORK 포기획 및 코딩 진행했던 작업물들을
    대분류로 분류하여 어떠한 경험과
    능력이 있는지 빠르게 파악이동

  • WORK SECTION - WORK LIST1More view 클릭시 NCS 과정을 통해 작업한
    프로젝트들이 상세하게 보여지면서
    각각의 기획 내용 PDF 확인

  • WORK SECTION - LIST2전직장 기획 작업물들만 보여지는
    섹션으로 이전 기획 경력을 어필

02. 디자인 완료한 작업물 그대로 마크업 하고, 기초 코딩 지식이 있는가?

직접 리뉴얼한 해피포인트 웹사이트와 다양한 직군의 웹페이지 마크업 진행 작업물을
전시함으로써 기본적으로 구현의 가능성을 어느정도 고려하여 기획 가능함을 보여주었습니다.

  • WORK SECTION - LIST1직접 리뉴얼 기획 및 디자인한 웹페이지와 동일하게
    코딩 구현 가능한 점과 그외 마크업한 웹페이지 전시

  • INDEX SEXTION - NCS 6개월간의 NCS 교육과정을 설명하고
    이에 맞는 직무 능력이 향상됨을 보여줌

03. 지원자의 성향이 묻어나고 인덱스에서 시선을 잡는 특별한 점이 있는가?

인덱스 비쥬얼에서 역동적인 애니메이션을 적용하고 나의 성격을 가장 직관적으로 보여 줄 수 있도록
좌우명을 최상단에 전시하고 선명한 컬러로 채용자에게 각인 될 수 있게 하였습니다.

  • INDEX SECTION - INTRO나의 성격이 묻어 나올 수 있게 좌우명 최상단 배치
    기획자에대한 의지를 보여주기 위해 소개글 작성
    선명하고 강렬한 컬러로 나를 각인

  • INDEX SECTION - ANIMATION역동적인 애니메이션 추가로 시선집중
